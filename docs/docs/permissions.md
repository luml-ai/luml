# Platform Permissions

## Обзор системы прав

В платформе реализована двухуровневая ролевая модель:

- **Organization level** — роль пользователя в организации определяет доступ к организационным ресурсам и служит «базовым» уровнем для работы с орбитами.
- **Orbit level** — роль пользователя внутри конкретного орбита применяется только тогда, когда его организационная роль не даёт доступа.

Приоритет: **Org Owner / Admin > Orbit Admin > Orbit Member**. Если пользователь является `owner` или `admin` организации, его orbit-роль игнорируется — он автоматически получает полный доступ к любому орбиту.

---

## Роли

### Organization Roles

| Роль | Описание |
|------|----------|
| `owner` | Единственный владелец организации. Полный доступ ко всему. Не может быть удалён или понижен. Назначается при создании организации и не может быть присвоен через приглашение. |
| `admin` | Широкий доступ: управление пользователями, орбитами, секретами, артефактами. Не может удалять организацию и не может назначать роль `admin` другим — это прерогатива `owner`. |
| `member` | Минимальный доступ: только чтение орбитов и возможность покинуть организацию. Реальный доступ к ресурсам внутри орбита — через orbit-членство. |

### Orbit Roles

| Роль | Описание |
|------|----------|
| `admin` | Полный контроль над орбитом и его ресурсами. |
| `member` | Работа с артефактами, коллекциями, деплойментами и спутниками в режиме чтения/создания/обновления, но без удаления и деплоя. |

## Алгоритм проверки прав

```
request(user, org_id, orbit_id?, resource, action)
│
├─ Нет членства в организации → 403 Insufficient Permissions
│
├─ Проверить org-роль пользователя
│   ├─ Право разрешено на уровне org → ✅ доступ
│   │
│   └─ Право НЕ разрешено на уровне org
│       ├─ orbit_id не указан → 403
│       │
│       └─ orbit_id указан → проверить orbit-роль
│           ├─ Нет членства в орбите → 403
│           ├─ Право разрешено на уровне orbit → ✅ доступ
│           └─ Право НЕ разрешено на уровне orbit → 403
```

**Ключевой принцип:** org `owner`/`admin` никогда не проверяются на уровне орбита. Они имеют доступ ко всем орбитам автоматически через org-level разрешения.

## Business Rules

Дополнительные ограничения, не отражённые в матрице разрешений:

### Управление пользователями организации

- Только `owner` может назначить роль `admin` (при приглашении и при обновлении роли).
- Никто не может назначить роль `owner` — она присваивается только при создании организации.
- Удалить `owner` из организации невозможно.
- Удалить организацию можно только если в ней остался единственный участник (сам `owner`).

### Управление приглашениями

- Принять (`accept`) или отклонить (`reject`) приглашение может только тот пользователь, чей email указан в приглашении.
- Нельзя пригласить пользователя с ролью `owner`.

### Управление участниками орбита

- Нельзя добавить себя в орбит — `create orbit_user` применяется только к другим пользователям.
- Нельзя изменить или удалить свою orbit-роль — только другие участники с правом `update`/`delete orbit_user` могут это сделать.

## Аутентификация и типы токенов

Помимо ролей, доступ к API ограничен типом токена:

| Тип токена | Префикс | Разрешённые маршруты |
|------------|---------|----------------------|
| JWT (сессия) | — | Все пользовательские маршруты, включая управление организацией |
| API Key | `dfs_` | Большинство ресурсных маршрутов (орбиты, артефакты, деплойменты и т.д.), **кроме** управления организацией и orbit-участниками |
| Satellite Key | `dfssat_` | Только внутренние маршруты для спутников (`/satellites/v1/*`) |

Управление организацией (участники, приглашения, настройки) доступно **только** через JWT — API-ключи не принимаются на этих маршрутах.

## Права в UI (Frontend)

Разрешения встроены в ответы API: каждый объект `Organization` и `Orbit` возвращается с полем `permissions` — словарём вида `{ resource: [action, ...] }`. Фронтенд использует эти данные для показа/скрытия элементов управления.

| Элемент UI | Условие отображения |
|------------|---------------------|
| Кнопка «New Orbit» | `org.permissions.orbit.includes("create")` |
| Кнопка настроек орбита | `orbit.permissions.orbit.includes("update")` |
| Кнопка «Delete Orbit» | `orbit.permissions.orbit.includes("delete")` |
| Кнопка «Delete Collection» | `orbit.permissions.collection.includes("delete")` |
| Кнопка «Delete Artifact» | `orbit.permissions.artifact.includes("delete")` |
| Тулбар обновления артефактов | `orbit.permissions.artifact.includes("update")` |
| Тулбар удаления артефактов | `orbit.permissions.artifact.includes("delete")` |
| Кнопка «Create Artifact» | `orbit.permissions.artifact.includes("create")` |
| Кнопка «Leave» организации | `org.permissions.organization.includes("leave")` |
| Ссылка на настройки организации | `org.permissions.organization.includes("read")` |
| Шестерёнка рядом с участником | Только для `member`-роли; для `admin` — только если текущий пользователь `owner` |
| Страница настроек организации | Недоступна для роли `member` (показывается заблокированный экран) |
